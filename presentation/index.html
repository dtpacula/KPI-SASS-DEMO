Defintion of Terms

  • CSS- CSS or Cascading Style Sheet describves how HTML elements should be displayed on screen.
  • SASS- SASS is a Style Sheet extension and scripting language that enhances CSS with variables, nestings, and inhertance.
  • Class- Attributes added to one more HTML elements to reference
  • ID- An ID is a single identifer to use on a single HTML Element

Syntactically Awesome Style Sheet a.k.a. SASS is a CSS preprocessor that extends CSS that allows developers to write code in a way that will comple to a CSS file.

SASS includes variables, functions, partial files, extensions/inhertiance, operators, and nesting that work together on compile to output a CSS file

People are applying certain CSS tools incorrectly.

CSS is inflexable and can be hard to maintain with an evergrowing file size. Example: Current GHX project CSS file is over 4000 lines.

vs. CSS

Why use Sass?

Cool Things Sass can Do!

Assign Basic elements like colors, fonts, etc. to variables for global use.

Import other SASS files to allow easy control. Elements or styling guides can be imported and used in other systems

Functions! SASS allows you to develop functions that can take paramaters to do all sorts of actions.

Build upon other declorations without declaring new class names.
